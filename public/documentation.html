myFlix



Objective

To build the server-side components of a "movies" web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sighn up, update their personal information, and create a list of their favorite movies.



Request URL HTTP Method Request body data format Response body data format
Return a list of all movies to user /movies GET None A JSON object holding data about all the movies
Return data (description, genre, director, image URL) about a single movie by title /movies/:title GET None A JSON object holding data about a single movie
{
id: 1,
title: The Shawshank Redemption,
description: ...,
genre: Drama,
director: Frank Darabont,
imgUrl: https://www.imdb.com/title/tt0111161/mediaviewer/rm1690056449/?ref_=tt_ov_i
}
Return data about a genre by name/title /movies/genre/:genreName GET None A JSON object holding data about a genre
{
id: 1,
name: Thriller,
description: ...
}
Return data about a director by name /movies/director/:name GET None A JSON object holding data about a director
{
id: 1,
name: Frank Darabont,
bio: ...,
birthYear: ...,
deathYear: ...
}
Register a user /users POST A JSON object holding data about the user to add, structured like
{
"name": ...,
"favoriteMovies": []
} 		A JSON object holding data about the user that was added, including an ID:
{
"id": "1",
"name": "...",
"favoriteMovies": []
}
Update user info /users/:id PUT None A text message indicating the username has been updated
Allow users to add a movie to thier list of favorites /users/:id/:movieTitle POST None A text message indicating that the movie has been added to favorites list
Allow users to remove a movie from their list of favorites /users/:id/:movieTitle DELETE None A text message indicating that the movie has been removed from favorites list
Allows existing users to deregister /users/:id DELETE None A text message indicating that the user has been removed
